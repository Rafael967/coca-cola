1 - O que é um repositório no Git e qual a sua importância?
R: É um local onde você armazena todos os arquivos e o histórico de alterações de um projeto.
É importante porque permite versionamento e colaboração.

2 - Qual a função do Git Bash?
R: É uma interface de linha de comando que permite usar comandos do Git no Windows.

3 - O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
R: Adiciona alterações para serem commitadas.
git add <arquivo> adiciona um arquivo específico, enquanto git add . adiciona todos os arquivos modificados.

4 - Por que utilizamos o ponto (.) no comando git add .?
R: O ponto (.) significa "todos os arquivos no diretório atual", 
então git add . adiciona todas as alterações no diretório.

5 - Explique o comando git commit e por que é importante criar uma mensagem descritiva.
R:  Salva as alterações no repositório com uma mensagem descritiva. 
A mensagem é importante para documentar o que foi alterado e por quê.

6 - Qual a função do comando git push?
R: Envia os commits do repositório local para um repositório remoto, como o GitHub.

7 - Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
R: Facilitam o rastreamento de alterações, ajudam a identificar bugs e permitem voltar a versões anteriores
 do código com mais facilidade.

8 - Explique a função das tags HTML e como elas estruturam uma página.
R: São elementos que estruturam o conteúdo de uma página, como <header>, <main>, <footer>, etc.
 Elas definem a organização e a semântica do conteúdo.

9 - O que são atributos em uma tag HTML e qual a sua função?
R: São informações adicionais dentro de uma tag, como href em ou src. 
Eles modificam ou configuram o comportamento da tag.

10- O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?
R: É uma ferramenta de desenvolvimento web que ajuda a criar código HTML, CSS e outro idiomas 
de marcação de fotma rápida e eficiente. Ele facilita o desenvolvimento de páginas HTML de várias maneiras: 
abreviações, autocompletar, snippets, personalização.

11- O que é o box-sizing no CSS e por que usamos ele no reset inicial?
R: Define como o tamanho dos elementos é calculado. Usamos box-sizing: border-box;
no reset para incluir padding e borda no tamanho total do elemento, evitando surpresas no layout.

12- Por que é importante utilizar tags semânticas no HTML?
R: Melhoram a acessibilidade e a organização do código, 
além de ajudar motores de busca a entenderem o conteúdo da página.

13- O que são media queries no CSS e qual a sua função?
R: Permitem aplicar estilos diferentes dependendo do tamanho da tela ou dispositivo,
criando designs responsivos.

14- Qual a função do GitHub Pages?
R: Hospeda sites estáticos diretamente de um repositório no GitHub,
facilitando a publicação de projetos.

15- O que é o Flexbox no CSS e para que ele é utilizado?
R: É um modelo de layout que facilita a organização e alinhamento de elementos em uma página,
especialmente em designs responsivos.

16- Qual a diferença entre justify-content e align-items em um container Flexbox?
R: justify-content alinha os itens ao longo do eixo principal (horizontal, por padrão), 
enquanto align-items alinha no eixo transversal (vertical, por padrão).

17- Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
R: Define como os itens dentro de um container Flexbox crescem para ocupar o espaço disponível.
Se um item tem flex-grow: 2, ele cresce duas vezes mais que os outros.

18- Explique a diferença entre flex-direction: column; e flex-direction: row;.
R: column organiza os itens verticalmente, enquanto row os organiza horizontalmente.
